## 💡 核心亮点速览

### 1️⃣ 多模态语义检索
- **CLIP统一编码**: 图像和文本映射到同一向量空间（512维）
- **跨模态匹配**: 支持以文搜图、以图搜图
- **零样本能力**: 无需领域训练，开箱即用

### 2️⃣ RAG增强检索链路
```
查询扩展 → 多路检索 → 结果融合 → AI解释 → 智能问答
```
- **查询扩展**: LLM生成多视角查询，召回率提升30%+
- **结果融合**: 去重+重排序，精选最相关结果
- **AI解释**: 自动生成结果说明，提升用户体验

### 3️⃣ 高性能向量检索
- **IVF-PQ索引**: 倒排+量化，内存压缩127倍
- **毫秒级响应**: 单查询<100ms（CPU）
- **GPU加速**: 支持Faiss GPU索引

### 4️⃣ Agent工具系统
- **标准化接口**: OpenAI Function Calling格式兼容
- **即插即用**: 4个内置工具（搜图、解释、问答）
- **LLM驱动**: 自然语言触发工具调用

### 5️⃣ 工程化设计
- **模块化架构**: 6大核心模块，职责清晰
- **配置驱动**: 功能开关控制，支持 graceful degradation
- **类型注解**: Python类型提示，代码可维护性高

### 6️⃣ 完整RAG Pipeline
| 阶段 | 技术 | 作用 |
|:---|:---|:---|
| 查询理解 | LLM Query Expansion | 生成多视角查询 |
| 向量检索 | CLIP + Faiss IVF-PQ | 语义相似匹配 |
| 结果处理 | De-duplication + Re-rank | 去重排序 |
| 生成增强 | LLM Explanation/QA | 解释+问答 |

## 🎤 面试话术速记

### 1分钟版本（电梯演讲）

```
这是一个基于CLIP+Faiss+LLM的多模态RAG检索引擎。

核心功能是：用自然语言搜索图片，或者用图片搜相似图。

技术亮点有三：
第一，用CLIP把图像和文本编码到同一向量空间，实现跨模态检索；
第二，用Faiss IVF-PQ索引优化，百万级数据毫秒级响应；
第三，用LLM做RAG增强，包括查询扩展、结果解释、智能问答。

Agent工具系统支持Function Calling，可以把检索能力集成到AI助手中。
```

### 3分钟版本（标准介绍）

```
这个项目是一个多模态RAG检索引擎，目标是实现"以文搜图、以图搜图"的智能检索。

【架构层面】
系统分三层：应用层用Flask提供Web和API；RAG引擎层负责查询扩展和结果生成；
检索层用CLIP做特征编码，Faiss做向量检索。

【核心创新点】
第一是多模态统一编码。CLIP把图像和文本都映射到512维向量空间，
这样"狗的图片"和"a dog"在向量空间是接近的，可以直接比较相似度。

第二是RAG完整链路。不是简单的检索，而是：
- 先用LLM把用户查询扩展成多个视角（比如"狗"扩展出"puppy"、"canine"）
- 多路检索后去重融合，提升召回率
- 最后用LLM生成结果解释和问答

第三是Agent工具化。把检索能力封装成标准工具，支持OpenAI Function Calling，
可以被外部Agent调用。

【工程亮点】
配置驱动设计，没配LLM也能跑，只是没有AI增强功能。
模块化设计，每个组件可独立测试。
```

## ⚠️ 面试注意事项

### ❌ 不要做
- **不要背诵代码**: 理解逻辑>记住代码，面试官问的是设计思想
- **不要夸大**: 承认项目还有改进空间（如增量更新、多租户）
- **不要只讲名词**: 每个技术术语都要能解释清楚原理
- **不要忽略工程**: 除了算法，还要讲模块化、配置管理

### ✅ 要做
- **画图讲解**: 随身带笔，画架构图、数据流图
- **举例子**: 用"搜索一只狗"的具体例子说明检索流程
- **讲权衡**: 为什么选择IVF-PQ而不是HNSW，为什么选CLIP而不是ResNet
- **谈改进**: 主动说"如果数据量更大，我会..."展示深度思考

### 📋 必须熟练的内容（面试前自查）

- [ ] CLIP的基本原理（对比学习、统一空间）
- [ ] Faiss IVF-PQ索引原理（倒排+量化）
- [ ] RAG流程（查询扩展→检索→融合→生成）
- [ ] Agent工具设计（Function Calling格式）
- [ ] 项目架构图（能徒手画出来）
- [ ] 至少一个技术决策的详细理由（如为什么选择IVF-PQ）
