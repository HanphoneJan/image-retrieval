## 1. 什么是多模态？

### 1.1 概念解释

**多模态 = 多种信息形态**

```
┌─────────────────────────────────────────────────────────────┐
│                      信息的不同形态                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   文字          图片           语音          视频           │
│   ┌───┐       ┌─────┐       ┌───┐        ┌─────┐          │
│   │文 │       │ 🖼️  │       │ 🔊 │        │ 🎬  │          │
│   │字 │       │     │       │   │        │     │          │
│   └───┘       └─────┘       └───┘        └─────┘          │
│                                                             │
│   "一只猫"    [猫的图片]    "miao~"      [猫的视频]        │
│                                                             │
│   ─────────────────────────────────────────────────────    │
│                                                             │
│   多模态 = 同时理解和处理这些不同类型的信息                    │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 生活中的例子

| 场景 | 涉及模态 | 说明 |
|:---|:---|:---|
| 看视频 | 画面 + 声音 + 字幕 | 同时处理三种信息 |
| 读绘本 | 文字 + 插图 | 图文结合理解故事 |
| 视频通话 | 图像 + 语音 | 看脸+听声识人 |

### 1.3 本项目的多模态

```
┌─────────────────────────────────────────────────────────────┐
│              本项目的多模态匹配                              │
│                                                             │
│        "一只金毛犬在草地上奔跑"                              │
│              │                                              │
│              │  CLIP模型处理                                 │
│              ▼                                              │
│        ┌─────────┐                                          │
│        │ 512维   │                                          │
│        │ 向量    │                                          │
│        │[0.1,   │                                          │
│        │ 0.5,   │                                          │
│        │ ...]   │                                          │
│        └────┬────┘                                          │
│             │                                               │
│             │  比较相似度                                    │
│             ▼                                               │
│        ┌─────────┐         ┌─────────┐                     │
│        │ 向量A   │ ←─────→ │ 向量B   │                     │
│        │(文字)   │  很相似  │(图片)   │                     │
│        └─────────┘         └─────────┘                     │
│             ↑                                               │
│             │                                               │

```

## 2. 什么是CLIP？

### 2.1 一句话解释

**CLIP 是一个能把文字和图片变成同一种"语言"（向量）的模型。**

### 2.2 类比理解

```
┌─────────────────────────────────────────────────────────────┐
│                   翻译官类比                                │
│                                                             │
│   英语            翻译官              中文                  │
│   "Dog"    →    [翻译]    →      "狗"                      │
│                                                             │
│   ─────────────────────────────────────────────────────    │
│                                                             │
│   CLIP就是这样的"翻译官"，但它更厉害：                       │
│                                                             │
│   文字            CLIP              向量空间                │
│   "狗"     →   [编码器]   →    [0.2, 0.8, ...]            │
│                                                             │
│   图片            CLIP              向量空间                │
│   [🐕]     →   [编码器]   →    [0.3, 0.7, ...]            │
│                                                             │
│   结果: "狗"和[🐕]在向量空间里很接近！                        │
│         可以用数字计算它们有多"像"！                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 训练过程（简化版）

```
┌─────────────────────────────────────────────────────────────┐
│              CLIP是如何学会"翻译"的？                        │
│                                                             │
│   训练数据：4亿张图片，每张图片都有一段描述文字               │
│                                                             │
│   ┌─────────────┐    ┌─────────────┐                       │
│   │ [猫的图片]  │    │ "一只橘猫   │                       │
│   │             │    │  在睡觉"    │                       │
│   └──────┬──────┘    └──────┬──────┘                       │
│          │                   │                              │
│          │   这对是正样本     │                              │
│          │   (应该相似)       │                              │
│          └─────────┬─────────┘                              │
│                    │                                        │
│                    ▼                                        │
│   ┌─────────────────────────────────────┐                  │
│   │           对比学习过程               │                  │
│   │                                     │                  │
│   │  目标：让正样本对的向量距离近        │                  │
│   │       让负样本对的向量距离远        │                  │
│   │                                     │                  │
│   │  正样本：图片 ↔ 它的文字描述         │                  │
│   │  负样本：图片 ↔ 其他随机文字         │                  │
│   │                                     │                  │
│   │  [猫图]───相近──→"一只橘猫在睡觉"   │                  │
│   │     ↘               ↗               │                  │
│   │       相远───────相远               │                  │
│   │     ↗               ↘               │                  │
│   │ [狗图]───相近──→"一只狗在奔跑"      │                  │
│   │                                     │                  │
│   └─────────────────────────────────────┘                  │
│                                                             │
│   学习后：相似的语义内容，向量也相似                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 为什么CLIP厉害？

| 能力 | 传统方法 | CLIP |
|:---|:---|:---|
| 理解语义 | 需要大量标注训练 | 开箱即用，理解自然语言 |
| 跨模态 | 做不到 | 文字和图片统一空间 |
| 泛化能力 | 只能识别训练过的类别 | 零样本识别新类别 |

**零样本例子**:

```
传统模型: 训练时看过1000种狗的图片 → 只能识别这1000种
CLIP:     即使没见过"柯基"的图片
          但知道"短腿"+"大屁股"+"狗" = 柯基
          → 能识别柯基！
```

## 3. 什么是向量检索？

### 3.1 概念类比

```
┌─────────────────────────────────────────────────────────────┐
│                   图书馆找书类比                            │
│                                                             │
│   传统检索（关键词匹配）                                     │
│   ─────────────────────                                     │
│   想找：关于"狗"的书                                        │
│   方法：翻看书名，找包含"狗"字的书                           │
│   问题：《犬类百科全书》被漏掉（没有"狗"字）                 │
│                                                             │
│   向量检索（语义匹配）                                       │
│   ─────────────────────                                     │
│   想找：关于"狗"的内容                                      │
│   方法：把所有书变成"内容向量"                              │
│         "狗"也变成向量                                      │
│         找向量最相近的书                                    │
│   结果：找到《犬类百科全书》《宠物饲养指南》《狗的故事》...  │
│                                                             │
│   神奇之处：不需要关键词完全匹配！                          │
│            能理解"狗"和"犬"是一个意思！                      │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 向量是什么？

```
┌─────────────────────────────────────────────────────────────┐
│                   向量 = 一串数字                            │
│                                                             │
│   "狗" → [0.1, 0.8, 0.3, 0.5, ..., 0.2]  (512个数字)       │
│                                                             │
│   可以把它想象成高维空间中的一个点：                         │
│                                                             │
│   2维空间（简单理解）：                                      │
│         y                                                   │
│         │                                                   │
│    "猫" ●                                                   │
│         │      ● "宠物"                                     │
│         │         ╲                                        │
│         │          ● "狗"                                   │
│         └────────────────→ x                                │
│                                                             │
│   实际上CLIP用512维空间，无法在纸上画出                      │
│   但道理一样：相似的东西在空间里距离近                       │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 相似度怎么算？

```
┌─────────────────────────────────────────────────────────────┐
│                   计算相似度的方法                           │
│                                                             │
│   方法1: 欧氏距离 (L2距离)                                   │
│   ─────────────────────────                                 │
│   想象两点间的直线距离                                       │
│                                                             │
│   A ●                                                       │
│     │＼                                                     │
│     │  ＼ 距离小 → 相似                                     │
│     │    ＼                                                 │
│     └──────● B                                              │
│                                                             │
│   公式: √(Σ(Ai - Bi)²)                                      │
│                                                             │
│   ─────────────────────────────────────────────────────    │
│                                                             │
│   方法2: 余弦相似度 (本项目实际使用)                         │
│   ─────────────────────────────────                         │
│   不看距离，看方向                                           │
│                                                             │
│         ↑                                                   │
│        ╱│╲   A和B方向相同 → 相似                            │
│       ╱ │ ╲                                                 │
│      ●──┼──●                                                │
│         │A  B                                               │
│                                                             │
│   公式: cos(θ) = (A·B) / (|A|×|B|)                         │
│                                                             │
│   本项目：L2归一化后，L2距离 ≡ 余弦相似度                    │
└─────────────────────────────────────────────────────────────┘
```

## 4. 什么是Faiss？

### 4.1 一句话解释

**Faiss 是一个超级快的"向量搜索引擎"，能在海量向量中快速找到最相似的。**

### 4.2 为什么需要Faiss？

```
┌─────────────────────────────────────────────────────────────┐
│                  暴力搜索 vs Faiss                           │
│                                                             │
│   场景: 在100万张图片中找最相似的10张                        │
│                                                             │
│   暴力搜索 (不用Faiss)                                       │
│   ─────────────────────                                     │
│   方法：计算查询图片与100万张图片的相似度                    │
│         排序，取前10                                        │
│   计算量：100万次距离计算                                   │
│   耗时：数秒 ~ 数十秒                                       │
│   问题：太慢！用户体验差！                                  │
│                                                             │
│   Faiss索引搜索                                              │
│   ─────────────────                                         │
│   方法：预先给100万张图片"分类整理"                         │
│         只搜索最可能相关的类别                             │
│   计算量：几千次距离计算                                    │
│   耗时：<100毫秒                                            │
│   加速：1000倍以上！                                        │
│                                                             │
│   类比：                                                     │
│   暴力搜索 = 在图书馆里一本本翻找                           │
│   Faiss    = 先查目录，直接去相关书架找                     │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 Faiss的"加速秘诀"

```
┌─────────────────────────────────────────────────────────────┐
│                   IVF 倒排索引                               │
│                                                             │
│   第一步：聚类                                               │
│   ───────────                                               │
│   把所有向量分成4096个"簇"（类）                            │
│                                                             │
│      簇1: 🐕🐩🦮 (都是狗)                                   │
│      簇2: 🐱🐈🐯 (都是猫)                                   │
│      簇3: 🚗🚕🚙 (都是车)                                   │
│      ...                                                    │
│      簇4096                                                 │
│                                                             │
│   第二步：查询                                               │
│   ───────────                                               │
│   1. 找查询向量属于哪个簇                                   │
│   2. 只搜索这个簇里的向量（可能还搜附近几个簇）              │
│   3. 从几万个候选中找到最相似的10个                         │
│                                                             │
│   计算量对比:                                               │
│   暴力：100万 × 512维 = 5亿次运算                          │
│   IVF：1次 + 244次 × 512维 ≈ 12万次运算                    │
│                                                             │
│   加速比: ~4000x                                            │
└─────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────┐
│                   PQ 乘积量化                                │
│                                                             │
│   问题：100万个512维向量，内存占用太大！                     │
│         100万 × 512 × 4字节 = 2GB                           │
│                                                             │
│   PQ解决方案：压缩存储                                       │
│   ─────────────────────                                     │
│                                                             │
│   原向量: [0.12, 0.53, 0.88, 0.33, ...]  (512个float32)    │
│      │                                                      │
│      │ 分割成32个子向量                                     │
│      ▼                                                      │
│   [16d][16d][16d]...[16d]                                   │
│      │   │   │      │                                       │
│      │   │   │      │ 每个子向量找一个最像的"代表"         │
│      ▼   ▼   ▼      ▼                                       │
│   code1 code2 code3 ... code32                              │
│      │   │   │      │                                       │
│      └───┴───┴──────┘                                       │
│            │                                                │
│            ▼                                                │
│   压缩后: 32个字节 (原来2048字节)                            │
│   压缩比: 64x                                               │
│                                                             │
│   内存占用: 100万 × 32字节 = 32MB (原来2GB)                  │
└─────────────────────────────────────────────────────────────┘
```

## 5. 什么是RAG？

### 5.1 一句话解释

**RAG = 先检索相关信息，再让AI基于这些信息回答问题。**

### 5.2 类比理解

```
┌─────────────────────────────────────────────────────────────┐
│                   开卷考试类比                              │
│                                                             │
│   闭卷考试 (传统AI)                                          │
│   ─────────────────                                         │
│   问题："2024年诺贝尔奖得主是谁？"                          │
│   AI：我的知识截止到2023年，不知道...                       │
│                                                             │
│   开卷考试 (RAG)                                             │
│   ─────────────────                                         │
│   步骤1: 去图书馆找资料                                     │
│          → 找到2024年诺贝尔奖新闻                           │
│                                                             │
│   步骤2: 带着资料回答问题                                   │
│          → "根据检索到的资料，2024年..."                    │
│                                                             │
│   好处：可以回答训练数据之外的新信息！                      │
│        回答有依据，不是瞎编！                                │
└─────────────────────────────────────────────────────────────┘
```

## 6. 什么是Agent？

### 6.1 一句话解释

**Agent = 能使用工具的AI，不只是聊天，还能执行动作。**

### 6.2 类比理解

```
┌─────────────────────────────────────────────────────────────┐
│                   助手 vs Agent                             │
│                                                             │
│   普通助手 (传统聊天AI)                                      │
│   ─────────────────────                                     │
│   你: "查一下明天北京的天气"                                 │
│   AI: "我无法查询实时天气信息，我的知识截止到..."            │
│   （只会动嘴，不会动手）                                     │
│                                                             │
│   Agent (工具使用AI)                                         │
│   ─────────────────                                         │
│   你: "查一下明天北京的天气"                                 │
│   AI思考: 用户要天气 → 我需要调用天气查询工具               │
│           → 调用get_weather(city="北京", date="明天")       │
│           → 获取结果：晴，25°C                              │
│   AI回答: "明天北京天气晴朗，气温25°C..."                    │
│   （会动嘴，也会动手！）                                     │
└─────────────────────────────────────────────────────────────┘
```

## 7. 技术术语速查表

| 术语 | 大白话解释 | 本项目中的作用 |
|:---|:---|:---|
| **向量/Embedding** | 把任何东西变成一串数字 | 图片和文字变成512个数字，可以比较相似度 |
| **相似度** | 有多像 | 数字越接近1，表示越相关 |
| **归一化** | 把数字缩放到统一范围 | 让不同向量能公平比较 |
| **索引** | 图书馆的目录系统 | 让搜索从100万次计算变成几千次 |
| **检索** | 查找 | 根据输入找到相似的图片 |
| **TopK** | 前K个结果 | 返回最相似的10张图片 |
| **召回率** | 找得多全 | 想找的100张里找到了80张，召回率80% |
| **精确率** | 找得多准 | 返回的10张里有8张相关，精确率80% |
| **Prompt** | 给AI的指令 | 告诉LLM怎么扩展查询、怎么解释结果 |
| **Token** | AI处理的最小单位 | "Hello"可能是1-2个token |
| **Inference** | 模型预测 | 把图片/文字送进模型得到结果 |

## 8. 常见问题Q&A

### Q1: 为什么用CLIP而不是其他模型？

**A**: CLIP把图片和文字放到同一个空间，这样"狗"的文字和图片可以直接比较。其他模型（如ResNet）只能处理图片，做不了文字搜图片。

### Q2: 向量检索和普通搜索有什么区别？

**A**: 普通搜索要关键词完全匹配，向量搜索理解语义。搜"狗"能找到标题带"犬"的图片，因为是同义词。

### Q3: Faiss为什么要用IVF-PQ，直接搜不行吗？

**A**: 数据少（几千张）可以直接搜。数据多（百万张）必须用Faiss加速，否则用户要等好几秒。

### Q4: RAG和直接问AI有什么区别？

**A**: AI可能"幻觉"（瞎编），RAG先找到真实资料再回答，更准确可靠。

### Q5: Agent和普通API有什么区别？

**A**: 普通API需要程序员写代码调用。Agent让AI自己决定调哪个API，普通用户说话就行。
